Frontend development involves creating the user interface and experience for web applications using HTML, CSS, and JavaScript. In this blog post, we will explore some of the JavaScript patterns commonly used in frontend development and how to implement them.

## MVC Pattern

The MVC (Model-View-Controller) pattern is a design pattern used to separate an application's concerns into three distinct components: the model, the view, and the controller. The model represents the data and business logic of the application, the view represents the user interface, and the controller acts as the intermediary between the two.

```javascript
class Model {
  constructor() {
    this.data = ["item 1", "item 2", "item 3"];
  }
  getData() {
    return this.data;
  }
}

class View {
  constructor() {
    this.list = document.getElementById("list");
  }
  render(data) {
    this.list.innerHTML = "";
    for (let i = 0; i < data.length; i++) {
      const li = document.createElement("li");
      li.textContent = data[i];
      this.list.appendChild(li);
    }
  }
}

class Controller {
  constructor(model, view) {
    this.model = model;
    this.view = view;
  }
  updateView() {
    const data = this.model.getData();
    this.view.render(data);
  }
}

const model = new Model();
const view = new View();
const controller = new Controller(model, view);
controller.updateView();
```

In the above example, we have created three classes for the model, view, and controller components of the MVC pattern. The model class contains the application's data, the view class contains the user interface, and the controller class acts as the intermediary between the two.

## Observer Pattern

The observer pattern is a design pattern where an object, called the subject, maintains a list of its dependents, called observers, and notifies them automatically of any state changes.

```javascript
class Subject {
  constructor() {
    this.observers = [];
  }
  addObserver(observer) {
    this.observers.push(observer);
  }
  removeObserver(observer) {
    const index = this.observers.indexOf(observer);
    if (index > -1) {
      this.observers.splice(index, 1);
    }
  }
  notifyObservers() {
    for (let i = 0; i < this.observers.length; i++) {
      this.observers[i].update();
    }
  }
  setState(state) {
    this.state = state;
    this.notifyObservers();
  }
}

class Observer {
  constructor(subject) {
    this.subject = subject;
    this.subject.addObserver(this);
  }
  update() {
    console.log(`Subject state updated: ${this.subject.state}`);
  }
}

const subject = new Subject();
const observer1 = new Observer(subject);
const observer2 = new Observer(subject);
subject.setState("new state");
```

In the above example, we have created a subject class and an observer class for the observer pattern. The subject maintains a list of observers and notifies them automatically of any state changes. The observer registers itself with the subject and is notified of any state changes.

## Factory Pattern

The factory pattern is a creational pattern that uses a factory function or class to create objects. The factory function or class is responsible for creating and returning instances of different object types based on the input parameters.

```javascript
class Circle {
  constructor(radius) {
    this.radius = radius;
  }
  draw() {
    console.log(`Drawing circle with radius ${this.radius}`);
  }
}

class Square {
  constructor(length) {
    this.length = length;
  }
  draw() {
    console.log(`Drawing square with length ${this.length}`);
  }
}

class ShapeFactory {
  create
  (shapeType, params) {
    switch (shapeType) {
      case "circle":
      return new Circle(...params);
      case "square":
      return new Square(...params);
      default:
      throw new Error(Invalid shape type: ${shapeType});
    }
  }
}

const shapeFactory = new ShapeFactory();
const circle = shapeFactory.create("circle", [5]);
circle.draw(); // Output: "Drawing circle with radius 5"
const square = shapeFactory.create("square", [10]);
square.draw(); // Output: "Drawing square with length 10"
```

In the above example, we have created two classes for the Circle and Square shapes and a factory class for creating instances of these shapes. The factory class uses a create method to create instances of different shape types based on the input parameters.
